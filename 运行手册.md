# SpecLoc 中文运行手册

本手册面向 Linux NVIDIA GPU 服务器。推荐使用仓库安装后的 `specloc` 统一命令；
`tools/` 和 `scripts/` 下的底层入口主要用于开发和排错。

## 1. 服务器要求

- Linux x86_64；
- NVIDIA GPU 与可用驱动；
- Python 3.10 或 3.11；
- Git；
- 可访问 PyTorch、OpenMMLab 和 ResNet-50 权重下载地址。

检查：

```bash
nvidia-smi
python3 --version
git --version
```

## 2. 克隆与安装

```bash
git clone https://github.com/DexZane/SpecLoc.git
cd SpecLoc
python3 -m venv .venv
source .venv/bin/activate
```

CUDA 12.1（RTX 4090 推荐）：

```bash
python -m pip install -r requirements.txt
```

CUDA 11.8：

```bash
python -m pip install -r requirements-cu118.txt
```

仅 CPU：

```bash
python -m pip install -r requirements-cpu.txt
```

不要执行 `pip install --upgrade setuptools`。项目固定使用
`setuptools==80.10.2`，因为 PyTorch 2.1.2 仍导入 `pkg_resources`，而
Setuptools 82 及之后版本已移除该模块。

安装后检查统一入口：

```bash
specloc --version
specloc info
```

## 3. RSOD 数据结构

把整个 RSOD 目录放到 `data/RSOD/`：

```text
data/RSOD/
├── images/                       976 张图像
├── annotations/
│   ├── instances_all.json
│   ├── instances_train.json      682 张
│   ├── instances_val.json        148 张
│   └── instances_test.json       146 张
├── splits/
├── images.sha256
├── manifest.json
└── README.md
```

如果数据在外部磁盘，可以创建软链接：

```bash
ln -s /mnt/data/RSOD data/RSOD
```

## 4. 一次完整的 RSOD 流程

训练前验收：

```bash
specloc doctor rsod
```

这个命令会依次完成：

1. 图像、标注、校验和、重复组和数据划分检查；
2. 精确核心依赖版本检查；
3. Git 工作树检查；
4. 配置加载和四类别模型构建；
5. CUDA GPU 检查。

开始标准单卡训练：

```bash
specloc train rsod
```

默认参数：

```text
GPU             1 张
AMP             开启
每卡 batch size 2
梯度累积        2
workers         2
输出目录        work_dirs/rsod_centernet_r50
```

训练结束后自动选择最新 `best_*.pth` 并评估：

```bash
specloc evaluate rsod
```

预测输出默认位于：

```text
work_dirs/rsod_centernet_r50/predictions.bbox.json
```

## 5. 断点续训与常用参数

```bash
# 自动恢复最近 checkpoint
specloc train rsod --resume

# 24 GB 显存充足时尝试 batch size 4
specloc train rsod --batch-size 4

# 关闭 AMP
specloc train rsod --no-amp

# 指定输出目录
specloc train rsod --work-dir work_dirs/rsod_trial_001

# 评估指定 checkpoint
specloc evaluate rsod --checkpoint /path/to/model.pth
```

高级 MMEngine 配置仍可覆盖，但日常运行不需要：

```bash
specloc train rsod --cfg-option train_cfg.max_epochs=2
```

## 6. 防止 SSH 断开

```bash
tmux new -s specloc
specloc train rsod
```

重新连接：

```bash
tmux attach -t specloc
```

另一终端监控显卡：

```bash
watch -n 2 nvidia-smi
```

## 7. AI-TOD-v2 正式基线

数据结构：

```text
data/AI-TOD/
├── train/
├── val/
└── annotations/
    ├── aitod_train_v2.json
    └── aitod_val_v2.json
```

记录实际授权来源和许可说明：

```bash
export AITOD_SOURCE='<实际授权下载地址或来源说明>'
```

运行：

```bash

```

RSOD 只负责工程验收；论文中的微小目标结论必须来自 AI-TOD-v2 等正式数据和预注册
分析流程。

## 9. 底层兼容命令

只有在调试 CLI 或 MMDetection 配置时才需要：

```bash
python tools/train.py configs/rsodCenternetR50.py --amp
python tools/test.py configs/rsodCenternetR50.py /path/to/model.pth
bash scripts/runCloudExperiment.sh --dataset rsod
```

正常用户不需要手工编写 `train_dataloader.*` 或
`optim_wrapper.accumulative_counts`。

## 10. 常见问题

### `No module named pkg_resources`

```bash
python -m pip install --force-reinstall 'setuptools==80.10.2'
```

### `No module named mmcv._ext`

不要从源码编译。重新使用与 CUDA 对应的 requirements：

```bash
# CUDA 12.1
python -m pip install --force-reinstall -r requirements.txt
```

### `torch.cuda.is_available() == False`

确认实例已分配 GPU，并检查：

```bash
nvidia-smi
specloc info
```

### CUDA out of memory

```bash
specloc train rsod --batch-size 1
```

CLI 会自动把梯度累积调整为 4，维持有效 batch size。

### 正式训练提示工作树不干净

```bash
git status
```

数据和 `work_dirs/` 已被忽略。若代码被修改，应提交或恢复明确版本后再训练，不要关闭
`require_clean_git`。

## 11. 输出与归档

每次正式运行至少保留：

```text
work_dirs/<run_id>/
├── experiment_metadata.json
├── <config_dump>.py
├── *.log / vis_data/
├── best_*.pth
└── predictions.bbox.json
```

元数据记录项目版本、Git commit、数据 manifest、随机种子、Python/PyTorch/MMDetection、
CUDA 和 GPU 信息。
